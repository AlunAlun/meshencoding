Mesh Encoding for the Web

Single mesh, 32-bit index buffer, 49 bits per vertex (71bpv with texture coords):

  • 11-bit/channel precision for vertices
  • 16-bit total for normals
  • 11-bit/channel for textures (where necessary)

We show results below with two delivery formats, UTF-8 (with modifications to allow 32-bit index buffers) and PNG image format.

Stamford Bunny

(4.9MB Wavefront OBJ; 1.67MB Raw Binary Mesh)

UTF+gzip. 227kB. 70ms decompression.

PNG. 306kB. 550ms decompression.

Comparison with WebGL-loader

Google's WebGL-loader is one of the better known mesh compression techniques for web rendering. Our UTF-based technique is similar, but improves on WebGL-loader in several ways:

  • It permits a 32-bit index buffer, overcoming issues with of the UTF-8 format which WebGL-loader does not deal with. This permits the mesh to be loaded (and displayed) in one go, and not as piecemeal chunks. this also makes it considerably less complicated to implement progressive meshes (watch this space!)
  • It compresses 3-channel normals to 2x8-bit channels, with no perceptible quality difference. This means that, for most meshes, it creates a smaller file size, and decompresses in the same time. The table below summarises the differences:
